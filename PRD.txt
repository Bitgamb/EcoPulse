You are a senior full-stack developer, UI/UX designer, database architect, and security-focused engineer.

Build a complete production-ready web application called “EcoPulse”.

Problem Statement:
Design a solution that helps individuals understand, track, and reduce their carbon footprint through simple actions and personalized insights.

The application must be polished, responsive, secure, accessible, efficient, and fully aligned with the problem statement. It should feel like a real SaaS product, not a static demo.

Main Goal:
EcoPulse allows users to log daily activities such as transport, electricity usage, food habits, shopping, and waste. The app calculates estimated carbon emissions in kg CO₂e, shows personalized insights, recommends simple actions, tracks goals, and motivates users through streaks, badges, and progress charts.

Tech Stack:
Frontend:
- Next.js 15
- TypeScript
- Tailwind CSS
- App Router
- Recharts for data visualization
- Lucide React for icons
- Framer Motion for subtle animations

Backend:
- Next.js API routes/server actions
- TypeScript business logic

Database:
- Supabase PostgreSQL

Authentication:
- Supabase Auth

Testing:
- Vitest
- React Testing Library

Deployment Target:
- Vercel

Important:
Do not build only static pages.
The app must have working forms, working calculations, protected routes, database integration, reusable components, loading states, error states, empty states, and seed/demo data.

DATABASE SCHEMA
--------------------------------------------------

Create Supabase SQL migration.

Tables:

profiles:
- id uuid primary key references auth.users(id)
- full_name text
- email text
- avatar_url text
- created_at timestamp default now()

carbon_entries:
- id uuid primary key default gen_random_uuid()
- user_id uuid references auth.users(id) not null
- category text not null
- activity_type text not null
- value numeric not null
- unit text not null
- co2_amount numeric not null
- entry_date date not null
- note text
- created_at timestamp default now()

eco_actions:
- id uuid primary key default gen_random_uuid()
- user_id uuid references auth.users(id) not null
- title text not null
- description text
- category text
- estimated_saving numeric default 0
- difficulty text
- completed boolean default false
- completed_at timestamp
- created_at timestamp default now()

goals:
- id uuid primary key default gen_random_uuid()
- user_id uuid references auth.users(id) not null
- title text not null
- category text
- target_reduction numeric not null
- current_progress numeric default 0
- deadline date not null
- status text default 'active'
- created_at timestamp default now()

badges:
- id uuid primary key default gen_random_uuid()
- user_id uuid references auth.users(id) not null
- badge_name text not null
- badge_description text
- unlocked_at timestamp default now()

user_stats:
- id uuid primary key default gen_random_uuid()
- user_id uuid references auth.users(id) not null
- total_co2_saved numeric default 0
- current_streak int default 0
- longest_streak int default 0
- eco_score int default 100
- last_activity_date date
- updated_at timestamp default now()

Enable Row Level Security for all tables.

Policies:
- Users can select, insert, update, and delete only their own rows.
- profiles table should allow user to read and update own profile.

Create database indexes:
- carbon_entries user_id
- carbon_entries entry_date
- eco_actions user_id
- goals user_id
- badges user_id
